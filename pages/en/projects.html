1. Business Problem

Manual exam grading prevents teachers from providing granular, personalized feedback due to a lack of time. Additionally, sending student data to external AI models raises major privacy and GDPR compliance issues.

2. Solution Provided

A Backend-as-a-Service assessment engine that instantly analyzes student reasoning anonymously, returning a structured diagnostic (misconceptions, grades, advice) directly to the teacher.

👉 Test Live Solution (SWA Interface)

3. How it Works

The API receives the exam and rubric, pseudonymizes personal information, queries powerful Large Language Models (LLMs) forcing a structured output format, and securely stores the diagnostic.

🔗 Source Code (GitHub Repo)

4. Technical Justifications

  • Azure Container Apps: A Serverless architecture that automatically scales to handle unpredictable traffic spikes during exam periods, while keeping costs under control.
  • Groq LPU (Llama 3.3): Utilizing specialized processors (LPUs) for ultra-fast inference at a fraction of the cost of traditional proprietary APIs.
  • Zod & TypeScript: Strictly necessary to validate the AI's data output format (Structured Outputs), preventing application crashes if the AI hallucinates or changes its response structure.

Speed & Batch Layers

1. Business Problem

In high-frequency algorithmic trading, reacting to volatility spikes in sub-second times is crucial, but maintaining a perfect, deduplicated history for model training is equally important. Fulfilling both needs often generates inconsistent data silos and prohibitive cloud infrastructure costs.

2. Solution Provided

A bifurcated data infrastructure (Lambda Architecture): a "Speed" layer for instant alerts, and a "Batch" layer for cold storage and perfect historical auditability.

3. How it Works

Live financial transactions are captured via WebSockets. The stream is split: one path triggers immediate volatility analysis via Serverless functions, while the other is stored massively "cold" before being deduplicated and consolidated.

🔗 Source Code (GitHub Repo)

4. Technical Justifications

  • AWS SQS FIFO (instead of Kinesis): A pure FinOps transition. SQS offers pay-per-use billing and guarantees strict ordering of financial messages, reducing cloud bills by 80% compared to Kinesis, which requires continuous resource allocation.
  • AWS Lambda (Speed Layer): Instantly triggers on each received message for ultra-fast volatility calculation (Bollinger Bands) while maintaining state in memory (Warm Start).
  • Snowflake & dbt (Batch Layer): Snowflake handles massive analytical storage, while dbt guarantees data idempotency (purging duplicates caused by network retries) using incremental models.
  • Stack: AWS (SQS, S3, Lambda) Python Async Snowflake dbt Core
