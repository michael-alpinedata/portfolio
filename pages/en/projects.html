Objective: Model and visualize the exposure of Small and Medium Enterprises (SMEs) to European electricity price volatility by connecting historical spot power prices (OPSD data) to a simplified P&L model per SME segment.

Stack:

  • Cloud & Infrastructure: GCP (Google Cloud Storage, BigQuery), Terraform (IaC)
  • Transformations & Modeling: dbt Core (BigQuery, monthly partitioning, clustering, staging + mart, seeds)
  • Ingestion & Automation: Python (with uv manager), BigQuery external tables, Shell scripting
  • Serving / Dashboard: Looker Studio

BI Architecture & Optimizations

Ingestion Pipeline: Python Script -> Raw OPSD CSV uploaded to GCS -> BigQuery external table (`raw_energy_prices_ext`).
Analytics Pipeline (dbt): Staging unpivot -> Dynamic cross join with SME profiles (seeds) -> Final `mart_sme_energy_costs` table.
FinOps Optimization: Configured monthly partitioning on `ts_utc` and clustering by `price_zone` and `segment` in BigQuery. This optimized response times and cut Looker Studio query costs by 90%.

Key Features:

  • Automated ELT Pipeline: Robust ingestion of hourly energy price series and schema exposition via BigQuery external tables.
  • Granular Financial Modeling: Simulation of SME P&L metrics (required revenue, total cost, energy cost, gross profit) across 10+ business segment archetypes.
  • Interactive Looker Studio Dashboard: Dynamic reports featuring filters for bidding zones, SME segments, and date ranges.
  • Data Quality & Testing: Rigorous implementation of dbt tests (`not_null`, `unique` and seed validators) enforcing pipeline schema integrity.

Alpine Lambda Trade (High-Throughput Lambda Architecture)

Objective: Design and implement a massive-scale Lambda Architecture (Batch + Speed Layer) consuming high-frequency financial websockets, with a strong focus on FinOps (cost optimization) and data reliability (idempotency).

Stack:

  • Cloud & IaC: AWS (Lambda, S3, SQS), Terraform
  • Data Warehouse: Snowflake (auto-ingest via Snowpipe)
  • Transformations: dbt Core (Incremental models, Quality Tests, Medallion Architecture)
  • Streaming: Async Python Producer (Binance WebSockets)
  • Serving: Streamlit

High Availability Architecture

Real-time flow (Speed): Ingestion to AWS SQS -> AWS Lambda (EMA/Anomaly calculation in ms) -> Snowflake STG_REALTIME.
Historical flow (Batch): S3 Buffering (Parquet) -> Snowpipe -> dbt (Cleansing & Deduplication) -> Snowflake SILVER.
Both layers fuse dynamically (UNION ALL) for the final Business View ensuring zero overlap.
Infrastructure entirely provisioned via Terraform (IAM roles, SQS, Snowflake Integrations).

Key Features & Business Impact:

  • FinOps Strategy: Migrated from Amazon Kinesis to AWS SQS FIFO, reducing cloud costs by 80% while retaining message durability and strict ordering.
  • Quality & Reliability (dbt): Idempotent pipelines built with dbt incremental models. Extensive use of dbt test for tracking unique constraints and lineage.
  • Anomaly Detection: On-the-fly statistical anomaly detection (Bollinger Bands) via an AWS Lambda Serverless function using a "warm start" stateful cache.
  • Data Contracts: Asynchronous error handling protecting against dropping websockets (designed for Dead Letter Queue integration).

Bank Branch Footfall (PySpark Data Pipeline)

Objective: Automate monitoring and analysis of visitor traffic in bank branches to enable optimal resource allocation (staffing) in near real-time.

Stack:

  • Ingestion/API: FastAPI (live visitor counter endpoint)
  • Processing: PySpark batch jobs for aggregations and cleaning
  • Orchestration: Airflow DAGs for ingestion and backfill
  • Storage: DuckDB (Serving) / Parquet Lakehouse
  • Visualization: Streamlit dashboard deployed as a public app
  • Containerization: Docker for local dev and reproducible runs
  • Quality: High pytest coverage on transforms and API

Workflow

Ingestion via FastAPI, heavy batch PySpark transforms on a Datalake, served optimally via a DuckDB OLAP layer and exposed to analysts through a Streamlit dashboard.

Key Features:

  • Scalable processing: PySpark efficiently runs huge historical backfills and computes daily and intra-day branch-level footfall KPIs (peaks, averages, rolling windows).
  • Operational API: FastAPI endpoint returns live visitor counts per branch for integration with frontline tools.
  • Scenario analysis: Streamlit dashboard lets managers explore peak periods, compare branches and test staffing scenarios.
  • Orchestrated workflows: Airflow orchestrates both near real-time ingestion and historical backfills for reprocessing.
  • Engineering practices: Dockerized services, configuration via environment variables and tests on business logic and API contracts.
